# Phase 6 — UI/UX 개선

전체 프론트엔드의 디자인 품질과 사용자 경험을 개선한다.

---

## 컬러 시스템

| 용도 | 색상 | 코드 |
|------|------|------|
| Background | 연한 라벤더 | `#FFEEFF` |
| Primary | 딥 네이비 | `#0C0A3E` |
| Secondary | 딥 퍼플 | `#7B1E7A` |
| Minor/Accent | 로즈 레드 | `#B33F62` |

파생 색상:
- Primary Light: `#1a1560` (hover 등)
- Secondary Light: `#9a3d99` (hover 등)
- Surface (카드/패널 배경): `#FFFFFF`
- Border: `#e0d8e8` (라벤더 톤 보더)
- Text Primary: `#0C0A3E`
- Text Secondary: `#555`
- Text Muted: `#999`

## 1. 글로벌 디자인 시스템

### 1-1. 타이포그래피
- 본문: 14px, line-height 1.6
- 제목: 16~18px, font-weight 600
- 캡션/라벨: 11~12px
- 코드: 'Consolas', 'Monaco', monospace

### 1-2. 컴포넌트 기본 스타일
- 버튼: border-radius 8px, primary/secondary/ghost 변형
- 입력 필드: border-radius 8px, focus 시 secondary 보더
- 카드: border-radius 12px, 가벼운 shadow
- 뱃지: border-radius 12px, pill 형태
- 드롭다운: 네이티브 select 대신 커스텀 드롭다운 컴포넌트

### 1-3. 아이콘
- 텍스트 기반 아이콘([~], [?] 등) 대신 이모지 또는 SVG 아이콘으로 교체

---

## 2. 사이드바 개선

### 2-1. 레이아웃
- 폭: 300px → 320px (문서명 잘림 방지)
- 탭: 하단 보더 스타일 → pill 형태 세그먼트 컨트롤
- 색상: 배경을 primary 또는 surface로, 탭 활성 상태를 secondary로

### 2-2. 문서 목록
- 상태 아이콘: [v], [x] 텍스트 → 컬러 도트 또는 이모지
- 파일 크기/업로드 시간 표시
- 문서 삭제 버튼 (현재 미구현)

### 2-3. 모델 관리
- Purpose 탭: 기본 HTML 버튼 → pill 토글 그룹, 활성 시 secondary 색상
- 모델 카드: 보더 카드 형태, 기본 모델에 secondary 좌측 보더
- 테스트 결과: 인라인 토스트 또는 배경색 피드백
- **모델 등록 시 purpose 다중 선택**: 체크박스 그룹으로 변경, 선택한 purpose마다 개별 레코드 생성

---

## 3. 채팅 영역 개선

### 3-1. 헤더
- 모델 선택: 네이티브 select → 커스텀 드롭다운
  - provider별 그룹 구분선 + 라벨
  - 선택된 모델 이름 + provider 아이콘 표시
  - 기본 모델에 뱃지 표시

### 3-2. 메시지 버블
- 사용자: secondary 배경 (기존 파랑 → `#7B1E7A`), 흰색 텍스트
- AI: surface 배경, primary 텍스트
- 최대 폭: 80% → 85%, 모바일 대응

### 3-3. Agent Steps
- 아이콘: 텍스트 → 이모지 또는 SVG
  - 질문 분석: 🔍, 행동 결정: 🤔, 문서 검색: 📄, 답변 생성: ✏️ 등
- 진행 중 표시: 마지막 step에 로딩 스피너 애니메이션
- 완료된 step: 약간 투명하게 (opacity 0.7)

### 3-4. 출처 표시
- 접힘/펼침 토글 (기본 접힘)
- 파일명 클릭 시 해당 청크 내용 모달 또는 확장 표시

### 3-5. Thumbs Up/Down 피드백
- 각 AI 메시지 하단에 👍 👎 버튼
- 클릭 시 서버에 피드백 전송 (기존 `/api/chat/feedback` API 활용)
- 선택된 상태 시각적 표시 (색상 변경)

### 3-6. 입력창
- 높이: 자동 확장 (textarea, 최대 5줄)
- 전송 버튼: 아이콘 버튼 (화살표)
- Enter 전송, Shift+Enter 줄바꿈

### 3-7. 마크다운 렌더링
- 코드 블록: 다크 테마 유지, 언어 라벨 표시
- 테이블: 짝수행 배경색을 라벤더 톤으로
- 인용문: secondary 색상 좌측 보더

## 4. 반응형 / 접근성

### 4-1. 반응형
- 모바일(768px 이하): 사이드바 접힘, 햄버거 메뉴로 토글
- 태블릿(768~1024px): 사이드바 축소 모드

### 4-2. 접근성
- 포커스 링: secondary 색상 outline
- 키보드 네비게이션: 모든 인터랙티브 요소 접근 가능
- aria-label 추가

---

## 5. 모델 등록 UX 개선

### 5-1. Purpose 다중 선택
- 현재: 단일 select → 1개 purpose만 선택
- 변경: 체크박스 그룹 → 여러 purpose 동시 선택
- 동작: 선택한 purpose마다 개별 `llm_model` 레코드 생성 (백엔드 배치 등록 API)
- UI: purpose별 체크박스 + 전체 선택 토글

### 5-2. 모델 등록 폼
- 단계형 폼: Provider 선택 → 모델 정보 입력 → Purpose 선택 → 확인
- Ollama: 자동 검색 결과에서 클릭으로 등록 (폼 자동 채움)

## 6. 기타

### 6-1. 토스트/알림
- 모델 테스트 성공/실패, 문서 업로드 완료 등의 피드백을 화면 상단 토스트로 표시

### 6-2. 로딩 상태
- 문서 업로드 중, 모델 검색 중 등에 스켈레톤 UI 또는 스피너

### 6-3. 빈 상태
- 문서 없을 때, 채팅 없을 때 일러스트 또는 가이드 메시지 개선

## 구현 순서

### Step 1: 컬러 시스템 + 글로벌 스타일
CSS 변수 정의, 기존 하드코딩 색상을 변수로 교체, 기본 컴포넌트 스타일 적용

### Step 2: 사이드바 + 모델 관리 UI
탭 디자인, 문서 목록 개선, 모델 카드 디자인, purpose 다중 선택

### Step 3: 채팅 영역
메시지 버블, agent steps 아이콘, 입력창 개선, 커스텀 모델 드롭다운

### Step 4: Thumbs Up/Down + 출처 토글
피드백 UI, 출처 접힘/펼침

### Step 5: 반응형 + 접근성 + 토스트
모바일 대응, 키보드 접근성, 피드백 토스트
