# Phase 16 — 표(Table) 지원

문서 내 표 데이터를 구조적으로 파싱하여 RAG 검색 및 응답에 활용한다.

---

## 1. 문서 파싱 시 표 추출

### 1-1. PDF 표 추출
- Apache PDFBox로 텍스트 추출 시 표 구조가 깨짐
- 표 감지 + 구조화 파싱 필요
- 방안 A: PDFBox의 텍스트 위치 정보로 행/열 재구성
- 방안 B: Tabula-java 라이브러리 활용 (PDF 표 전문 추출)
- 방안 C: LLM에게 표 영역을 마크다운 표로 변환 요청

### 1-2. Markdown 표
- 이미 마크다운 표 형식 (`| col1 | col2 |`)은 텍스트로 파싱됨
- 청킹 시 표를 분리하지 않도록 표 경계 감지 추가

## 2. 표 데이터 청킹

### 2-1. 표 전용 청크
- 표는 하나의 청크로 유지 (분할하면 의미 손실)
- 표 앞뒤 문맥(캡션, 설명)을 함께 포함
- 청크 메타데이터에 `type: TABLE` 표시

### 2-2. 표 요약
- 큰 표의 경우 LLM으로 요약 텍스트 생성
- 요약 텍스트를 별도 청크로 저장 → 검색 정확도 향상

## 3. 응답 시 표 렌더링

### 3-1. 마크다운 표 출력
- LLM 응답에 표가 포함된 경우 마크다운 표로 출력
- 프론트엔드 react-markdown + remark-gfm이 이미 표 렌더링 지원

### 3-2. 출처에 표 표시
- 소스 표시 영역에서 표 청크는 표 형태로 렌더링
