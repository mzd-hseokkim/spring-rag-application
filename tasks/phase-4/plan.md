# Phase 4 — Advanced Enhancement (선택)

RAG 품질을 한 단계 끌어올리기 위한 고도화 기능. 필요에 따라 선택적으로 도입한다.

---

## 1. Contextual Chunking

### 1-1. 컨텍스트 보강 청킹
- 문서 인덱싱 시 각 청크에 LLM으로 문서 전체 맥락 요약을 첨부
- 청크 단독으로도 문맥 파악 가능 → 검색 정확도 향상
- 인덱싱 비용 증가 (청크당 LLM 호출) vs. 검색 품질 개선 트레이드오프

### 1-2. 계층적(Parent-Child) 청킹
- 작은 청크로 검색 정밀도 확보, 매칭 시 상위(부모) 청크 반환
- 부모 청크가 더 풍부한 컨텍스트를 LLM에 제공

> **처리 방식**: 비동기 — 문서 인덱싱 파이프라인 내 처리

---

## 2. Agentic RAG

### 2-1. 적응적 검색 (Adaptive Retrieval)
- LLM이 검색 필요 여부를 스스로 판단
- 검색 불필요 시 파라메트릭 지식으로 직접 응답
- 불필요한 검색 호출 30~40% 절감

### 2-2. 멀티스텝 추론
- 복합 질문을 하위 질문으로 분해
- 각 하위 질문에 대해 독립적으로 검색 → 응답
- 중간 결과를 종합하여 최종 답변 합성

### 2-3. 도구 사용 (Tool Use)
- LLM이 벡터 검색, SQL 조회, 외부 API 호출 중 적절한 도구를 선택
- Spring AI Function Calling 활용

> **처리 방식**: 동기 — 에이전트 루프 내에서 반복적 판단/실행

---

## 3. HyDE (Hypothetical Document Embeddings)

### 3-1. 가상 문서 생성
- LLM이 질의에 대한 가상의 이상적 답변을 생성
- 생성된 가상 답변을 임베딩하여 검색
- 모호하거나 추상적인 질의에 효과적

> **처리 방식**: 동기 — 쿼리 변환 단계

---

## 도입 판단 기준

| 기능 | 도입 조건 |
|------|----------|
| Contextual Chunking | 검색 정확도가 기대치 미달일 때 |
| Agentic RAG | 복합 질문 비율이 높거나, 다중 데이터 소스 연동이 필요할 때 |
| HyDE | 추상적/모호한 질의가 많을 때 |
