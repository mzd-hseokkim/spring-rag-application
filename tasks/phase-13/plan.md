# Phase 13 — 임베딩 배치 최적화 + 청크 전략 튜닝

문서 처리 성능을 개선하고, 청크 품질을 높여 RAG 검색 정확도를 향상시킨다.

---

## 1. 임베딩 배치 최적화

### 1-1. 현재 문제
- 청크마다 개별 임베딩 API 호출 → N개 청크 = N번 호출
- 대용량 문서 처리 시 병목

### 1-2. 개선
- 배치 임베딩: 여러 청크를 묶어서 한 번에 임베딩 요청 (배치 크기 설정 가능)
- 병렬 배치: 배치 단위로 병렬 처리 (CompletableFuture)
- 진행률 추적: 임베딩 진행 상황을 SSE로 전달

## 2. 청크 전략 튜닝

### 2-1. 청크 크기 최적화
- 현재 시맨틱 청킹의 min/max 파라미터 검토
- 문서 유형별 최적 청크 크기 실험 (PDF vs Markdown vs Text)
- 관리자 설정으로 청크 파라미터 조정 가능하게

### 2-2. 오버랩 전략
- 청크 간 오버랩 비율 조정
- 문맥 유실 방지를 위한 슬라이딩 윈도우 개선

### 2-3. 메타데이터 보강
- 청크에 문서 제목, 섹션 헤더 정보 포함
- 검색 시 메타데이터 활용으로 정확도 향상

## 3. 재인덱싱
- 관리자 메뉴에서 "재인덱싱" 기능 추가
- 청크 전략 변경 후 기존 문서를 새 전략으로 재처리
