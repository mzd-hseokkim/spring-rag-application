# Phase 18 — AI 문서 생성 (Document Generation)

사용자 입력(제안요청서, 주제, 데이터 등)을 기반으로 AI가 구조화된 문서를 생성한다.
템플릿 기반으로 일정 수준 이상의 품질을 보장하며, PDF/PPTX/HTML 포맷으로 출력한다.

---

## 핵심 원칙

**AI는 콘텐츠만 생성하고, 포맷팅은 템플릿 엔진이 담당한다.**

- AI가 HTML/CSS를 직접 생성하면 품질이 불안정함
- AI → 구조화된 데이터(JSON) → 템플릿 엔진 → 최종 문서 구조로 분리
- Spring AI의 `.entity()` 구조화 출력을 활용하여 안정적인 JSON 생성

---

## 아키텍처 개요

```
┌─────────────────────────────────────────────────────────────┐
│                    사용자 입력                                │
│  (텍스트, 파일 업로드, 채팅 대화 내용 등)                       │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─ Phase 1: PLAN ─────────────────────────────────────────────┐
│  사용자 요청 + RAG 컨텍스트 분석                               │
│  → AI가 문서 아웃라인(섹션 구조) 생성                           │
│  → Spring AI .entity() → DocumentOutline record              │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─ Phase 2: GENERATE ─────────────────────────────────────────┐
│  섹션별 개별 AI 호출                                          │
│  → 각 섹션마다 RAG 검색 + 전용 프롬프트                        │
│  → 구조화된 콘텐츠(제목, 본문, 핵심 포인트, 출처) 생성           │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─ Phase 3: REVIEW (선택적) ──────────────────────────────────┐
│  AI 자체 검증: 완성도, 정확성, 톤 체크                         │
│  → 품질 미달 시 해당 섹션만 재생성                              │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─ Phase 4: RENDER ───────────────────────────────────────────┐
│  구조화된 데이터 → 템플릿 엔진 → 최종 파일                      │
│  - HTML/PDF: Thymeleaf + OpenHTMLtoPDF                       │
│  - PPTX: Apache POI XSLF + 마스터 템플릿                     │
└─────────────────────────────────────────────────────────────┘
```

---

## 구현 우선순위

1. **MVP**: 텍스트 입력 → AI 구조화 생성 → HTML 출력/다운로드
2. **PDF 지원**: HTML → PDF 변환 추가
3. **템플릿 관리**: DB 기반 템플릿 CRUD + 어드민 UI
4. **PPTX 지원**: POI 기반 PPTX 렌더링
5. **고급 기능**: 리뷰 단계, 사용자 편집/재생성, 품질 대시보드
