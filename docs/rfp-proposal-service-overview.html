01

서비스 구성

하나의 플랫폼에서 문서 기반 질의응답, 제안서 자동 생성, 예상 질의문 작성까지 — 제안 업무에 필요한 세 가지 핵심 기능을 통합 제공합니다.

Service 1
RAG 챗봇
업로드된 문서를 벡터 검색하여 근거 기반 답변을 실시간 스트리밍으로 제공합니다. 웹검색 폴백과 멀티스텝 추론을 지원합니다.
핵심: 벡터 검색 + LLM 생성 + 소스 인용
Service 2
제안서 생성
고객 RFP를 분석하여 목차를 추출하고, 요구사항을 매핑한 뒤, 섹션별 콘텐츠를 자동 생성합니다. 5단계 위자드 방식입니다.
핵심: RFP 분석 → 목차 → 매핑 → 생성 → 렌더링
Service 3
예상 질의문 생성
고객 RFP와 제안서를 비교 분석하여 Gap을 식별하고, 평가위원 페르소나별 예상 질문과 답변을 자동 생성합니다.
핵심: Gap 분석 + 페르소나 + Q&A 자동 생성
02

RAG 챗봇

사용자의 질문을 분석하고, 업로드된 문서에서 관련 내용을 벡터 검색하여 근거 있는 답변을 실시간으로 생성합니다. 복잡한 질문은 자동으로 분해하여 다단계 추론을 수행합니다.

STAGE 1
Analyze
질문 분석
쿼리 압축 + 행동 결정
+ 하위 질문 분해
STAGE 2
Search
문서 검색
벡터 DB 유사도 검색
웹검색 자동 폴백
STAGE 3
Generate
답변 생성
문서 컨텍스트 기반
SSE 스트리밍 응답
OUTPUT
Response
응답 전달
마크다운 렌더링
소스 인용 표시
🔍
멀티스텝 추론
복잡한 질문을 하위 질문으로 자동 분해하여 반복 검색합니다. 단순 키워드 매칭이 아닌 의미 기반 추론을 수행합니다.
🌐
웹검색 폴백
내부 문서에서 충분한 답변을 찾지 못하면 Tavily 웹검색으로 자동 전환됩니다. 사용자가 직접 토글할 수도 있습니다.
📂
문서 필터링
Tag와 Collection 기반으로 검색 대상 문서를 선택할 수 있습니다. 특정 프로젝트나 분야의 문서만 대상으로 질의할 수 있습니다.
분석 결과에 따른 자동 분기
DIRECT_ANSWER
검색 없이 직접 답변 가능한 일반 질문
SEARCH
벡터 DB에서 관련 문서 검색 후 답변 생성
WEB_SEARCH
내부 문서 부족 시 웹검색으로 자동 전환
CLARIFY
질문이 모호할 때 사용자에게 명확화 요청
03

제안서 생성

고객 RFP(제안요청서)를 업로드하면 AI가 자동으로 목차를 추출하고, 요구사항을 매핑한 뒤, 참조 문서와 웹검색을 활용하여 섹션별 콘텐츠를 생성합니다. 5단계 위자드로 전 과정을 안내합니다.

STEP 1
Draft
작업 생성
템플릿 · 고객 문서
참조 문서 선택
STEP 2
Analyze
목차 추출
RFP 분석 → 트리 구조
사용자 편집 가능
STEP 3
Map
요구사항 매핑
요구사항 → 섹션 매핑
Gap Matrix 시각화
STEP 4
Generate
콘텐츠 생성
RAG + 웹검색 기반
섹션별 자동 작성
STEP 5
Render
문서 렌더링
PDF · PPTX · HTML
다운로드 · 미리보기
🌳
목차 트리 에디터
AI가 추출한 목차를 드래그&드롭으로 재배치하고 섹션을 추가/삭제할 수 있습니다. Markdown 내보내기도 지원합니다.
📊
Gap Matrix
고객 요구사항과 목차 섹션 간 매핑 현황을 시각화합니다. 미배치 요구사항은 새 섹션을 자동 생성하여 누락을 방지합니다.
✏️
섹션별 편집 & 재생성
각 섹션을 개별 수정하거나 사용자 지시사항을 포함해 재생성할 수 있습니다. 진행 상황은 SSE로 실시간 표시됩니다.
출력 형식
PDF PPTX HTML 미리보기 Markdown
04

예상 질의문 생성

고객 RFP와 제안서를 비교 분석하여 Gap을 식별하고, 평가위원 페르소나(기술위원, 사업관리위원 등)별로 맞춤형 예상 질문과 모범 답변을 자동 생성합니다.

PHASE 3

Gap 분석 모드
RAG 벡터 검색 모드
제안서에서 요구사항별 관련 내용을 벡터 유사도 검색으로 빠르게 매칭합니다. 대량 처리에 적합합니다.
LLM LLM 정밀 분석 모드
LLM이 제안서 전체를 읽고 정밀하게 충족/부분충족/미충족을 판정합니다. 더 정확하지만 시간이 더 소요됩니다.
페르소나 시스템
평가위원의 직책과 관심 영역을 페르소나로 정의합니다.

각 페르소나는 자신의 전문 분야에서 Gap이 큰 부분을 중점적으로 질문합니다. 기본 제공 페르소나 외에 사용자 정의 페르소나를 추가할 수 있습니다.

예시: 기술위원, 사업관리위원, 보안전문가, PM 등

05

공통 기반 기능

📄
문서 관리
PDF · DOCX · HWP 등 다양한 형식의 문서를 업로드하면 자동으로 청크 분할 → 벡터화 → 인덱싱됩니다. Tag와 Collection으로 분류 관리합니다.
🤖
멀티모델 관리
용도별로 다른 LLM을 설정할 수 있습니다. 채팅 · 쿼리 분석 · 문서 생성 등 목적에 따라 최적의 모델을 선택합니다.
📊
대시보드 & 모니터링
토큰 사용량, 응답 품질 평가, 사용 통계를 대시보드에서 실시간 모니터링합니다. 감사 로그로 전체 사용 이력을 추적합니다.
🔒
인증 & 권한
JWT 기반 인증과 역할 기반 접근 제어(USER / ADMIN)를 제공합니다. 관리자 전용 페이지와 기능이 분리되어 있습니다.
⚡
비동기 파이프라인
문서 인제스팅, 콘텐츠 생성 등 시간이 오래 걸리는 작업은 비동기로 처리되며 SSE를 통해 진행 상황을 실시간으로 알립니다.
💾
캐싱 & 최적화
요구사항 추출 결과, 문서 분석 결과를 캐싱하여 반복 작업의 속도를 향상합니다. Redis를 활용한 대화 이력 캐싱도 포함됩니다.
06

기술 스택

Backend
  • Java 21 + Spring Boot 3.4
  • Spring AI 1.1 — ChatClient 기반 LLM 통합
  • Spring Data JPA + Flyway 마이그레이션
  • Redis — 대화 이력 캐싱 (TTL 60분)
  • SSE — 실시간 스트리밍 응답
Frontend
  • React 19 + TypeScript 5.9
  • Vite 8 — 빌드 도구
  • TanStack Query — 서버 상태 관리
  • shadcn/ui — UI 컴포넌트
  • Markdown 렌더링 지원
Database & Search
  • PostgreSQL 16 — 관계형 데이터 저장
  • pgvector — 벡터 임베딩 저장 · 유사도 검색
  • Tavily — 웹검색 API 통합
Build & Deploy
  • Gradle (Kotlin DSL) — 백엔드 빌드
  • pnpm — 프론트엔드 패키지 관리
  • Docker Compose — PostgreSQL + pgvector

Data Flow

문서 업로드 → 청크 분할 → 임베딩 벡터화 → pgvector 저장
사용자 질의 → 쿼리 분석 → 벡터 유사도 검색 → LLM 생성 → SSE 스트리밍
제안서 생성 → RFP 분석 → 목차 추출 → 요구사항 매핑 → 섹션 생성 → PDF/PPTX 렌더링
질의문 생성 → Gap 분석 → 페르소나별 Q&A 생성 → Excel 출력
