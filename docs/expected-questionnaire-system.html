How It Works

세 단계로 완성되는 질의서

문서 업로드부터 Excel 다운로드까지, AI가 전체 파이프라인을 자동으로 처리합니다.

1

문서 업로드 & 분석

과제 문서를 업로드하면 6가지 관점으로 폭넓게 분석합니다.

  • 사업개요, 기술, 수행방안, 관리, 인력, 유지보수 6가지 관점으로 검색
  • Map-Reduce 방식으로 대량 문서도 분산 처리: 배치별 부분 분석 후 통합
  • 분석 결과는 Redis에 캐싱 (24시간 TTL), 같은 문서 재사용 시 즉시 처리
2

페르소나 기반 질문 생성

다양한 평가 관점에서 깊이 있는 질문을 생성합니다.

  • 기본 4개 페르소나: 고객(발주자), 기술평가위원, 가격평가위원, 경영평가위원
  • 사용자가 커스텀 페르소나 추가 가능 — AI가 전용 시스템 프롬프트 자동 생성
  • 각 페르소나마다 개별 LLM 호출로 깊이 있는 질문 생성
  • 질문에는 난이도(상/중/하), 평가영역 카테고리, 출처 포함
3

다중 소스 활용 & 출력

참조 문서와 웹 검색을 활용해 답변 품질을 극대화합니다.

  • 참조 문서: 유사 제안서, 기존 질문집으로 답변 품질 향상
  • 웹 검색(Tavily): 업계 동향, 기술 트렌드로 심층 질문 추가
  • 출력: Excel(.xlsx) 다운로드 + HTML 미리보기
  • 출처 구분: "과제문서:", "참조문서:", "웹:", "누락:" 접두어

Key Features

핵심 기능

깊이 있는 질의서를 만들기 위한 6가지 핵심 기술을 제공합니다.

🔍

Map-Reduce 분산 분석

대량 문서를 배치로 나눠 분석합니다. 10,000자 단위 분할 후 부분분석, 그리고 통합. 토큰 한도 걱정 없이 어떤 크기의 문서도 처리합니다.

👤

AI 페르소나 프롬프트

이름/역할/관심분야만 입력하면 AI가 전용 시스템 프롬프트를 자동 생성합니다. 수동 편집 및 재생성도 가능합니다.

🌐

Tavily 웹 검색 통합

체크박스 하나로 외부 정보를 활용합니다. 페르소나별 관심분야 기반 검색 쿼리를 자동으로 생성합니다.

📦

Redis 분석 캐싱

같은 문서 조합이면 분석 결과를 재사용합니다. SHA-256 해시 기반 캐시 키, 24시간 TTL 자동 만료로 효율적으로 관리합니다.

📊

구조화된 출력

Excel 파일에 페르소나별 시트, 난이도 색상 구분, 출처를 명시합니다. HTML 미리보기도 동시에 생성됩니다.

🔄

실시간 진행 상태

SSE(Server-Sent Events)로 모든 단계를 실시간 표시합니다. 문서 수집에서 배치 분석, 페르소나별 검색/생성까지 전 과정을 추적합니다.

Architecture

아키텍처 상세

문서 분석부터 질문 생성까지, 시스템의 핵심 설계 원리를 살펴봅니다.

문서 전체를 보고 질문합니다

기존 RAG 방식은 검색 쿼리에 매칭되는 청크만 보고 질문을 만듭니다. 이 시스템은 6가지 평가 관점으로 문서를 폭넓게 수집한 뒤, LLM이 전체 구조를 분석합니다.

  • 분석 결과에서 "약점", "누락 사항"을 중심으로 질문 소재를 도출
  • 문서에 없는 항목(예: 재해복구 계획 누락)도 질문 대상으로 포착
  • 2단계 파이프라인: 분석(Analyzing) 후 생성(Generating)으로 분리
  • 분석 단계의 결과물이 생성 단계의 입력이 되는 구조
Step 1: Analyzing
6가지 관점 검색 → 청크 수집
Map: 배치별 부분 분석
Reduce: 통합 분석 (강점/약점/누락)
Step 2: Generating
페르소나 × (분석결과 + 참조 + 웹)
질문/답변 생성

질문 대상과 참조 자료를 분리합니다

과제 문서와 참조 문서는 역할이 다릅니다. 시스템은 두 유형을 명확히 구분하여 처리하고, 출처를 투명하게 기록합니다.

  • 과제 문서: 제안요청서, 제안서 등 질문을 생성할 대상
  • 참조 문서: 유사 제안서, 기존 질문집, 업계 자료 등 품질 향상용
  • 과제 문서는 Map-Reduce 분석 대상, 참조 문서는 페르소나별 RAG 검색 대상
  • 두 출처를 구분하여 sources에 "과제문서:", "참조문서:" 접두어로 기록

답변은 핵심 포인트 나열입니다

구어체/대화체가 아닌 핵심 포인트 나열 형식으로 모범답변을 생성합니다. 발표 준비에 바로 활용할 수 있는 구조화된 형태입니다.

  • "- Circuit Breaker 적용 범위 제시\n- 임계값 수치 근거..." 형태의 핵심 포인트
  • Bloom's Taxonomy 기반 난이도 분포: 하 20%, 중 50%, 상 30%
  • 평가 영역 6가지: 사업이해도, 기술적합성, 수행방안, 프로젝트관리, 인력/실적, 지원/유지보수
  • 각 질문-답변 쌍에 난이도, 카테고리, 출처를 명시
Question
제안서에 언급된 마이크로서비스 아키텍처의 장애 전파 방지 전략이 구체적으로 어떻게 구현되는지 설명해 주십시오.
상 기술적합성
Answer
- Circuit Breaker 패턴 적용 범위 및 임계값 설정 근거 제시
- 서비스 간 비동기 통신(메시지 큐) 도입 계획
- Bulkhead 패턴으로 장애 격리 범위 명시
- 장애 시나리오별 Fallback 전략 수립
