# v1.0 - 2026-03-25
사용자가 업로드한 문서가 있는 RAG 시스템입니다.
아래 대화 이력, 문서 목록, 질문을 종합 분석하세요.

[대화 이력]
%s

[업로드된 문서 목록]
%s

[사용자 질문]
%s

아래 3가지를 한 번에 판단하세요:

1. 대화 맥락을 반영하여 독립적 검색 쿼리를 작성
2. 적절한 행동 선택: SEARCH, DIRECT_ANSWER, CLARIFY, WEB_SEARCH
3. SEARCH인 경우 하위 질문 분해 필요 여부 판단

판단 기준:
- 문서와 조금이라도 관련되면 SEARCH (기본값)
- DIRECT_ANSWER는 인사/감사/일반 대화에만 사용
- CLARIFY는 질문이 너무 모호할 때만
- WEB_SEARCH는 다음 경우에 사용:
  - 최신 뉴스, 시사, 실시간 정보가 필요한 질문
  - 업로드된 문서로 답변할 수 없는 일반 지식/기술 질문
  - 문서 목록이 비어있고 구체적인 정보가 필요한 질문
  - 문서와 무관한 외부 정보가 필요한 질문
- 하위 질문은 최대 3개까지만

응답 형식 (정확히 따라야 합니다):
ACTION: SEARCH 또는 DIRECT_ANSWER 또는 CLARIFY 또는 WEB_SEARCH
QUERY: (독립적 검색 쿼리)
DOCUMENTS: (대상 문서 ID, 쉼표 구분. 전체 검색이면 ALL)
SUB_QUERIES: (없으면 NONE, 있으면 한 줄에 하나씩)
