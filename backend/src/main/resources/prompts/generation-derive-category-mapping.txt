## 🎯 임무

### 임무 1: 카테고리 → leaf 매핑
각 카테고리가 어느 leaf(들)에 배치되어야 하는지 결정하세요.

**규칙**:
1. 하나의 카테고리는 1개 또는 여러 leaf에 매핑 가능 (1:N 허용)
2. 기술/기능 카테고리(기능, 성능, 데이터, 인터페이스 등)는 기술 성격 leaf에 매핑
3. 관리/운영 카테고리(프로젝트 관리, 품질, 보안관리 등)는 해당 성격 leaf에 매핑
4. leaf title과 상위 경로를 보고 가장 의미적으로 가까운 leaf를 선택
5. **의미상 겹치는 leaf가 여러 개면**, 관점(임무 2)에 따라 분할 — 하나의 카테고리를 관점별로 다른 leaf에 배치

### 임무 2: leaf의 관점 역할 부여
각 leaf에 관점 역할(role)을 부여하세요. 이 role은 Enricher가 content를 생성할 때 사용됩니다.

**관점 코드**:
| 코드 | 의미 | 적용 대상 |
|---|---|---|
| WHAT | 기능·사용자 경험 | 기능·사용자 인터페이스 관련 섹션 |
| HOW-tech | 기술 구현·아키텍처 | 기술·아키텍처·인프라 관련 섹션 |
| WHY | 전략·비전 | 전략·목표·차별화 관련 섹션 |
| HOW-method | 방법론·절차 | 방법론·일정·추진 관련 섹션 |
| OPS | 운영·유지보수 | 운영·유지보수·모니터링 관련 섹션 |
| MGMT | 관리·통제 | 프로젝트관리·위험관리·거버넌스 관련 섹션 |
| CTRL-tech | 기술적 통제 | 기술 구현으로서의 보안·품질 (암호화·SQL 인젝션 등) |
| CTRL-mgmt | 관리적 통제 | 프로세스로서의 보안·품질 (교육·서약서·감사 등) |
| MISC | 기타·미분류 | 다른 어느 role에도 해당 안 되는 섹션 |

**규칙**:
1. 각 leaf에 정확히 하나의 role 부여
2. 의미상 겹치는 sibling leaf(예: 같은 주제를 다른 각도로 다루는 두 섹션)는 **반드시 다른 role** 부여
3. 기타사항/미분류 섹션은 "MISC"

### 임무 3: 카테고리 → leaf 매핑 시 role 고려

같은 카테고리가 여러 leaf에 매핑될 수 있을 때, role에 따라 분할:
- 예: "보안" 카테고리가 기술 섹션과 관리 섹션 양쪽에 필요하면, 기술 섹션(role=CTRL-tech)과 관리 섹션(role=CTRL-mgmt) 양쪽에 매핑
- 예: "기술 구현" 카테고리가 기능 섹션과 적용기술 섹션 양쪽에 연관이 있으면, WHAT role leaf와 HOW-tech role leaf로 분할

## 출력 형식
반드시 아래 JSON 객체 형식으로만 응답하세요 (JSON 외 다른 텍스트·설명 없이).

```
{{
  "categoryToLeafKeys": {{
    "<카테고리명>": ["<leafKey1>", "<leafKey2>"],
    ...
  }},
  "leafKeyToRole": {{
    "<leafKey1>": "<관점 코드>",
    "<leafKey2>": "<관점 코드>",
    ...
  }}
}}
```

### 추상 출력 예시 (도메인 무관)

```
{{
  "categoryToLeafKeys": {{
    "기능": ["2.1"],
    "성능": ["2.3"],
    "데이터": ["2.2"],
    "인터페이스": ["2.4"],
    "보안": ["3.1", "4.2"],
    "품질": ["4.1"],
    "관리": ["4.3"]
  }},
  "leafKeyToRole": {{
    "1.1": "WHY",
    "1.2": "WHY",
    "2.1": "WHAT",
    "2.2": "HOW-tech",
    "2.3": "HOW-tech",
    "2.4": "WHAT",
    "3.1": "CTRL-tech",
    "4.1": "MGMT",
    "4.2": "CTRL-mgmt",
    "4.3": "MGMT"
  }}
}}
```

이 예시에서 "보안" 카테고리는 role=CTRL-tech인 3.1과 role=CTRL-mgmt인 4.2에 분할 매핑되었습니다. 각 leaf는 보안을 서로 다른 관점에서 다룹니다.
