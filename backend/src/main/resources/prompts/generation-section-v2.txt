# v2.1 - 2026-03-24
다음 섹션의 콘텐츠를 작성하세요.

규칙:
- content는 마크다운 형식으로 작성하되, 3000자 이내로 핵심 내용을 서술하세요.
- content 안에 마크다운 표(|---|--- 형식)를 절대 사용하지 마세요. 표 데이터는 반드시 tables 배열에 넣으세요.
- 데이터 비교, 수치, 일정 등은 tables 배열을 적극 활용하세요.
- highlights는 핵심 포인트 3~5개를 간결하게 작성하세요.
- tables의 각 항목은 caption(표 제목), headers(열 제목 배열), rows(행 배열의 배열) 형식입니다.
- 표의 행은 최대 8개로 제한하세요.
- references에는 참고한 자료의 출처를 기록하세요. 없으면 빈 배열로 두세요.
- **layoutType**: 아래 유형 중 이 섹션에 가장 적합한 것을 하나 선택하세요.
- **layoutData**: 선택한 layoutType에 맞는 구조화된 데이터를 제공하세요 (아래 설명 참고).
- 반드시 아래 JSON 형식으로만 응답하세요. 마크다운 코드블록이나 설명 없이 순수 JSON만 출력하세요.

## 레이아웃 유형 (layoutType)
- TEXT_FULL: 전체 너비 본문. content만 사용, layoutData 불필요.
- TEXT_TABLE: 상단 본문 + 하단 표. content + tables 사용, layoutData 불필요.
- COMPARE_2COL: 2단 비교 (AS-IS/TO-BE 등). layoutData에 leftLabel, leftBody, rightLabel, rightBody 필요.
- PROCESS_HORIZONTAL: 수평 단계별 프로세스. layoutData에 steps 배열 필요 (각 step: label, description).
- KPI_CARDS: 핵심 수치 카드 3~4개. layoutData에 cards 배열 필요 (각 card: value, label, description).
- TABLE_FULL: 전체 너비 표 중심. tables 배열 사용, content는 간략 설명만.
- TEXT_IMAGE: 텍스트 + 다이어그램. content가 텍스트, layoutData에 imagePlaceholder(다이어그램 설명), imageCaption 필요.
- IMAGE_FULL: 전체 너비 다이어그램. layoutData에 imagePlaceholder(다이어그램 상세 설명), imageCaption, description 필요.

## 선택 가이드
- 서론, 개요, 방침 → TEXT_FULL
- 설명 + 데이터표 → TEXT_TABLE
- 현행/제안, 장단점 비교 → COMPARE_2COL
- 절차, 방법론, 프로세스 → PROCESS_HORIZONTAL
- 기대효과, 성과 수치 → KPI_CARDS
- 일정표, 인력표, 스펙표 → TABLE_FULL
- 아키텍처 설명 + 구성도 → TEXT_IMAGE
- 전체 시스템 구성도, 네트워크 토폴로지 → IMAGE_FULL
