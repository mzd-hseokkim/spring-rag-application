# v2.2 - 2026-03-24
다음 섹션의 콘텐츠를 작성하세요. 이 문서는 **프레젠테이션형 제안서**입니다.

## 핵심 원칙
- 이 문서는 발표용 제안서이므로, **장문의 텍스트 서술을 최소화**하고 **시각적 요소를 극대화**하세요.
- 각 섹션(장표)에는 **governing message**(핵심 메시지)를 반드시 포함하세요. 이는 해당 장표가 전달하려는 한 줄 요약입니다.
- content는 **간결한 bullet point 중심** (3~5개). 한 bullet은 1~2줄 이내. 문장형 서술 금지.
- **TEXT_FULL은 정말 서론/개요에만 사용**하세요. 대부분의 섹션은 시각적 레이아웃을 사용해야 합니다.
- **완성본이 아닌 초안**이라는 관점에서 작성하세요. "이 위치에 ○○ 다이어그램 삽입" 같은 시각 요소 가이드를 적극 포함하세요.

## 규칙
- governingMessage: 이 장표의 핵심 메시지 (1문장, 20자 내외). 장표 상단에 강조 표시됩니다.
- content는 마크다운 형식, 1000자 이내. bullet point 중심으로 간결하게.
- visualGuide: 이 장표에 포함해야 할 시각 요소를 기술하세요 (예: "시스템 아키텍처 3-tier 구성도", "AS-IS/TO-BE 비교 인포그래픽", "연간 일정 Gantt 차트"). 없으면 빈 문자열.
- content 안에 마크다운 표(|---|--- 형식)를 절대 사용하지 마세요. 표 데이터는 반드시 tables 배열에 넣으세요.
- highlights는 핵심 포인트 3~5개를 간결하게 작성하세요.
- tables의 각 항목은 caption(표 제목), headers(열 제목 배열), rows(행 배열의 배열) 형식입니다.
- references에는 참고한 자료의 출처를 기록하세요. 없으면 빈 배열로 두세요.
- **layoutType**: 아래 유형 중 이 섹션에 가장 적합한 것을 하나 선택하세요.
- **layoutData**: 선택한 layoutType에 맞는 구조화된 데이터를 제공하세요.
- 반드시 아래 JSON 형식으로만 응답하세요. 마크다운 코드블록이나 설명 없이 순수 JSON만 출력하세요.

## 레이아웃 유형 (layoutType)
- TEXT_FULL: 전체 너비 본문. **서론/개요에만 사용. 다른 섹션에는 사용 금지.**
- TEXT_TABLE: 상단 본문 + 하단 표. content + tables 사용.
- COMPARE_2COL: 2단 비교 (AS-IS/TO-BE 등). layoutData에 leftLabel, leftBody, rightLabel, rightBody 필요.
- PROCESS_HORIZONTAL: 수평 단계별 프로세스. layoutData에 steps 배열 필요 (각 step: label, description). 3~6단계.
- KPI_CARDS: 핵심 수치 카드 3~4개. layoutData에 cards 배열 필요 (각 card: value, label, description).
- TABLE_FULL: 전체 너비 표 중심. tables 배열 사용, content는 간략 설명만.
- TEXT_IMAGE: 텍스트 + 다이어그램. content가 텍스트, layoutData에 imagePlaceholder(다이어그램 상세 설명), imageCaption 필요.
- IMAGE_FULL: 전체 너비 다이어그램. layoutData에 imagePlaceholder(다이어그램 상세 설명), imageCaption, description 필요.

## 레이아웃 선택 가이드 (반드시 이 가이드를 따르세요)
- 서론, 사업 개요 → TEXT_FULL (유일하게 허용)
- 현황 분석, 문제점 → COMPARE_2COL (현행 vs 개선)
- 아키텍처, 시스템 구성 → TEXT_IMAGE 또는 IMAGE_FULL
- 방법론, 프로세스, 절차 → PROCESS_HORIZONTAL
- 기대효과, 성과 → KPI_CARDS
- 일정, 인력, 스펙 → TABLE_FULL
- 기능 목록, 요구사항 대응 → TEXT_TABLE
- 기술 비교, 솔루션 비교 → COMPARE_2COL 또는 TABLE_FULL

## 비율 목표
