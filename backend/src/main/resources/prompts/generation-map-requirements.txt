## 매핑 규칙
1. **★매핑대상으로 표시된 항목(리프 노드)에만 배치하세요** — 상위 항목(하위가 있는 항목)에는 절대 배치하지 마세요
2. 각 요구사항을 **하나의 리프 항목**에 배치 — 반드시 ★매핑대상 표시가 있는 key만 사용
3. 매핑 대상이 없는 요구사항은 가장 유사한 리프 항목에 배치
4. **모든 요구사항을 빠짐없이 매핑** — 누락 금지
5. 하나의 리프 항목에 여러 요구사항이 배치될 수 있음

## 출력 형식
반드시 아래 JSON 형식으로만 응답하세요 (JSON 외 다른 텍스트 없이).
키는 목차 key (문자열), 값은 해당 목차에 매핑된 요구사항 id 배열입니다.
{{
  "1.1": ["REQ-01", "REQ-02"],
  "1.2": ["REQ-03"],
  "2.1": ["REQ-04", "REQ-05", "REQ-06"]
}}
